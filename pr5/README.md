# Практична номер 5

## Завдання за варіантом (10)

### Опис завдання

Побудувати випадок, де збережений вказівник “всередину” блоку призводить до неможливості коректного звільнення базової адреси.

### Опис рішення

*Тут буде описано чому ця проблема в цілому виникає.*

Менеджер пам'яті зберігає дані блоку безпосередньо перед адресою, яку він 
видає нам як вказівник. Коли викликати функцію **free(ptr)** йде відступ на 
певну кілокість байтів назад щоб прочитати ці дані - і якщо передавати адресу 
всередину блоку, то ці дані не прочитаються. В результаті він прочитає вказані 
нами дані як необхідні, спробує виконати некоректну операцію - і результатом буде 
*core dumped*.

### Виконання програми

```
malloc addr: 0x5990985252a0
Вказівник всередину: 0x5990985252a4 (зсув: 4)

free(inside_ptr)
free(): invalid pointer
Aborted (core dumped)
```
